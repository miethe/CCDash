## 3. Top recommendations

### R1 — Ingest `attachment` events (S, low risk, high impact)
14 distinct `attachment.type` values observed in 427 lines:

- `hook_success` (52): `hookName`, `hookEvent`, `toolUseID`, `stdout`, `stderr`, `exitCode`, `durationMs`, `command`. This is the *post-hook outcome* — distinct from the existing `progress.hook_progress` pre-event.
- `task_reminder` (84): `content`, `itemCount`. TodoWrite snapshot injected at user-message boundaries.
- `command_permissions` (46): `allowedTools[]`. Permission allowlist snapshot.
- `diagnostics` (38): `files[]`, `isNew`. IDE lint/typecheck injection.
- `skill_listing` (37): `content`, `skillCount`, `isInitial`. Skill roster snapshot.
- `nested_memory` (45): `path`, `displayPath`, `content`. Child `CLAUDE.md` injection.
- `deferred_tools_delta` (31) and `mcp_instructions_delta` (23): tool catalog / MCP diffs (`addedNames`, `removedNames`, `readdedNames`).
- `ultrathink_effort` (10): marker that ultrathink was engaged for next turn.
- `file` (8): user-attached file (`filename`, `displayPath`, `content`).
- `queued_command` (7): `prompt`, `commandMode` (`"prompt"` or `"task-notification"`).
- `edited_text_file` (5), `opened_file_in_ide` (3), `selected_lines_in_ide` (2): IDE-context injection with `filename`, `lineStart`/`lineEnd`, `ideName`, `displayPath`.

**Plug-in point**: new branch in `parser.py` after the `summary` branch (~L2628, before `custom-title`). Append `system` logs with `eventType="attachment:<subtype>"`; for `hook_success`/`file`/`nested_memory`/IDE-context, also `add_artifact()`. **Effort: S; Risk: low** (additive).

### R2 — Surface `attributionSkill` / `attributionPlugin` (S, low risk, high analytics value)
- Evidence: `attributionSkill` on 742 assistant lines (`"dev:execute-phase"`, `"dev:execute-contract"`, `"planning"`); `attributionPlugin` on 188 (`"dev"`).
- Today `AgentSession.skillsUsed` only fills when a `Skill` tool is *invoked* — but skills auto-loaded via command bindings never produce a `Skill` tool call, so those usages are invisible. Attribution fields are the canonical source.
- Plug-in: `record_entry_context` (L2219), add Counters; roll into session at L1857-area assembly. Add `AgentSession.pluginsUsed: list[str]`. **Effort: S; Risk: low.**

### R3 — Promote `sessionKind`, `promptId`, `leafUuid`, `bridgeSessionId` to session metadata (S–M, low risk)
- `sessionKind="bg"` (66 entries) marks TaskCreate background threads — CCDash today has no way to distinguish foreground from background sessions.
- `promptId` (1,276 entries) changes across resumed sessions — useful for "long resumed thread" detection.
- `bridge-session` events (`cse_*` IDs) appear when a Codex Session Exchange is active.
- Plug-in: add `sessionKind`, `bridgeSessionId`, `lastSequenceNum` to `AgentSession` in `backend/models.py:154`; mirror in `types.ts`. **Effort: M; Risk: low** (additive).

### R4 — Capture `last-prompt` and `ai-title` events (S, low risk)
- `ai-title` is Claude-Code-generated (`"debug-sync-constraint-violation"`) — more reliable than the current heuristic that pulls from first user message.
- `last-prompt` carries verbatim final prompt + `leafUuid` pointing into the JSONL — ideal for a "resume from here" affordance.
- Plug-in: branches at L2630. Add `titleSource` provenance ("ai-title" | "first-message" | "manual") to avoid silently displacing existing titles. **Effort: S; Risk: low.**

### R5 — Promote `system.subtype` to a first-class taxonomy (M, low risk)
- `turn_duration` carries `durationMs` + `messageCount` per turn (one CCDash sample: 49s, 532s, 56s) — today only used to mark sessions terminal.
- `away_summary` carries a model-authored session-state summary — the single most valuable session-resume aid for the Session Inspector.
- `bridge_status` (3) marks bridge transitions; `local_command` (20) carries slash-command names.
- Plug-in: when `entry_type == "system"` (currently never branched), dispatch on `subtype`. Aggregate `turn_duration` into `AgentSession.turnDurations[]`. **Effort: M; Risk: low.**

### R6 — Persist `thinking` block `signature` (XS, low risk)
- Every `thinking` block has `signature` (294/294). Parser at L2918-2920 reads `block.get("thinking")` text only. Signature is the redacted-thinking attestation token. **Effort: XS.**

### R7 — Frontend visualization of nested usage breakouts (M, low risk)
- Parser already extracts `service_tier`, `inference_geo`, `speed`, `server_tool_use.{web_fetch_requests,web_search_requests}`, `cache_creation.ephemeral_{1h,5m}_input_tokens`, `iterations` (`parser.py` L1672-1700+).
- `types.ts` only exposes flat `cacheCreationInputTokens` / `cacheReadInputTokens`; `components/SessionInspector.tsx:2469` only renders the merged sum.
- Plug-in: extend `types.ts` + add SessionInspector sub-panel. **Effort: M; Risk: low** (display-only).

### R8 — `permission-mode` transition events (S, low risk)
- 75 lines; observed `"bypassPermissions"` correlates with `sessionKind=="bg"`. The dedicated event is the *transition*; the field on other lines is the *state*.
- Plug-in: branch + `AgentSession.permissionModeTransitions[]`. **Effort: S.**

### R9 — Tool-category enrichment for new built-in tools (S, low risk)
- `TaskCreate`, `TaskUpdate`, `TaskList`, `Monitor`, `EnterWorktree`, `ExitWorktree`, `AskUserQuestion`, `ToolSearch`, `SendMessage`, `NotebookEdit` all appear; only `Skill` and `Task/Agent` get a `toolCategory` today.
- Plug-in: extend classification near L3066-3140. **Effort: S; Risk: low** (category labels become UI facets — choose deliberately).

### R10 — `attributionPlugin` rollup widget
Spin-off of R2 powering a top-plugins-by-session dashboard widget.
